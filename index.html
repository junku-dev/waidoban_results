Waidoban Results

About

This is a web page designed to show the capabilities of waidoban.

Don't know what waidoban is? Check out the github.

Briefly, waidoban is a CLI that acts as a wrapper for qpdf and ghostscript. The goal was to make a tool that would make it easy to manipulate, compress, and optimize multiple PDF files at once.

Conclusions

Based on my observations, compressing a file and then using optimize to reduce images or scans to 150 DPI is optimal in MOST CASES. 150 DPI is pretty crisp and the file size is much smaller. Especially if you are viewing the file on a smaller device where the benefits of higher resolutions and higher DPIs are less noticable.

Using a combination of "optimize" and "compress" may yield VERY desirable results, but this is situaional as it really depends on the nature of the file. If the PDF file contains images and text then using both "optimize" and "compress" should work well. If your PDF files comprises of mainly image scans, like manga, then "compress" may not do much and you will see the best gains using "optimize". If your PDF files comprises of text, the "compress" will yield the best results.
